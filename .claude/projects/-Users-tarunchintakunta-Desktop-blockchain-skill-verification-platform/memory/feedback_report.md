Supervisor Vikas Sahni gave the following feedback on the research report:

1. **No literature review in the introduction** — Intro should state the problem and research question only. All specific paper citations belong in Section 2 (Related Work).

2. **Abstract must be short (<200 words)** — No features, no tech stack, no function names. Frame as research investigation, not software development. Current version: 125 words.

3. **Research contributions must not include tech stack** — No "Next.js 14, TypeScript, Drizzle ORM" etc. Those go in an Implementation section. Contributions should describe research outcomes (pipeline, algorithm, evaluation framework).

4. **Karatas 2018 — be careful** — Do NOT claim blockchain skill verification "only exists after 2018" or that Karatas was "the earliest". Vikas is sure earlier work exists and will challenge this. Current fix: "an early exploration of this space" (neutral phrasing).

5. **Methodology should not be a tech stack dump** — Some tech can be mentioned but not a full table listing every library. Describe phases in research terms. Tech stack table was removed.

**Why:** Vikas will raise questions on these specific points during the viva. Getting them wrong loses marks.

**How to apply:** Any future report writing must follow these rules. Abstract < 200 words, no tech in contributions, no lit review in intro, careful with "earliest/first" claims about any reference.
